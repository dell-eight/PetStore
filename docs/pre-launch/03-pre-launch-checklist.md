# WagTrail Final Pre-Launch Checklist

Use this checklist before sending paid traffic or announcing the store.

## Design

- [ ] Homepage feels like a real ecommerce brand, not only a concept.
- [ ] Hero clearly communicates pet walk/travel accessories.
- [ ] Product section is easy to scan.
- [ ] Trust section includes specific, truthful claims.
- [ ] Footer exists and includes business links.
- [ ] Visual style is consistent across sections.
- [ ] No placeholder icons or fake UI actions remain.

## Copy

- [ ] Hero headline is approved.
- [ ] Subheadline clearly explains Hydrate, Clean, Carry.
- [ ] CTA labels are specific.
- [ ] Product descriptions are benefit-led.
- [ ] Product objections are answered.
- [ ] FAQ copy exists.
- [ ] Footer copy exists.
- [ ] Placeholder legal/business copy is clearly marked for owner review.

## Mobile

- [ ] Tested at 360px.
- [ ] Tested at 375px.
- [ ] Tested at 390px.
- [ ] Tested at 414px.
- [ ] Tested at tablet width.
- [ ] No horizontal scrolling.
- [ ] CTA buttons are easy to tap.
- [ ] Header/menu works.
- [ ] Product cards are readable.
- [ ] Animations do not feel heavy.

## SEO

- [ ] Page title includes brand and product category.
- [ ] Meta description includes key products.
- [ ] One clear `h1`.
- [ ] Product sections use proper headings.
- [ ] Image alt text is descriptive.
- [ ] Product keywords appear naturally.
- [ ] Open Graph metadata exists.
- [ ] Internal links work.
- [ ] FAQ content is crawlable.

## Accessibility

- [ ] Keyboard focus is visible.
- [ ] Menu can be used with keyboard.
- [ ] Links do not use empty `#` destinations.
- [ ] Buttons have labels.
- [ ] Images have alt text.
- [ ] Contrast is checked.
- [ ] Motion reduction is respected.
- [ ] Color is not the only indicator.

## Performance

- [ ] Hero image is compressed.
- [ ] WebP or AVIF image variant exists.
- [ ] Below-fold images lazy load.
- [ ] Build passes.
- [ ] Bundle size is reviewed.
- [ ] No unnecessary dependencies.
- [ ] No large unused assets.
- [ ] No layout shift from images.

## Product Pages

- [ ] TrailSip product page or detail section exists.
- [ ] PawPure product page or detail section exists.
- [ ] TrailPack product page or detail section exists.
- [ ] Product dimensions are listed.
- [ ] Materials are listed.
- [ ] Care instructions are listed.
- [ ] Product photos are added.
- [ ] Price and currency are clear.
- [ ] CTA goes to cart/checkout or approved prelaunch flow.

## Policies

- [ ] About page exists.
- [ ] Contact page exists.
- [ ] FAQ page exists.
- [ ] Shipping policy exists.
- [ ] Return/refund policy exists.
- [ ] Privacy policy exists.
- [ ] Terms of service exists.
- [ ] Owner has reviewed legal/business copy.

## Payments

- [ ] Ecommerce platform selected.
- [ ] Shopify Buy Button or Storefront API decision made.
- [ ] PayMongo decision made if needed.
- [ ] Test payment completed.
- [ ] Failed payment behavior tested.
- [ ] Refund process tested.
- [ ] Secure checkout message visible.

## Analytics

- [ ] Google Analytics added.
- [ ] Meta Pixel added if running Meta ads.
- [ ] TikTok Pixel added if running TikTok ads.
- [ ] CTA click events tracked.
- [ ] Product click events tracked.
- [ ] Checkout click events tracked.
- [ ] Purchase event tracked after checkout integration.
- [ ] Privacy policy covers tracking.

## Testing

- [ ] `npm run build` passes.
- [ ] Local preview checked.
- [ ] Desktop checked.
- [ ] Mobile checked.
- [ ] Header links checked.
- [ ] Footer links checked.
- [ ] Product CTAs checked.
- [ ] Contact email checked.
- [ ] Test order completed.

## Deployment

- [ ] Vercel project connected.
- [ ] Environment variables configured.
- [ ] Domain connected.
- [ ] HTTPS active.
- [ ] Production build checked.
- [ ] Social share preview checked.
- [ ] Final owner approval received.
